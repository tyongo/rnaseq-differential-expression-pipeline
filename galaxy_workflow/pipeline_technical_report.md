Draft Technical Report: RNA-seq Read Processing & Quantification Pipeline 					

Overview

This repository documents a reproducible bioinformatics pipeline designed to process raw Next-Generation Sequencing (NGS) data into gene-level expression counts. The workflow was architected using Galaxy to ensure accessibility and reproducibility. The pipeline transforms raw FASTQ reads from a mouse mammary gland study (Basal vs. Luminal cells) into a count matrix suitable for differential expression analysis.
Pipeline Architecture & Methodology
1. Data Acquisition & Integrity (Input)

	• Data Source: Raw sequencing reads (FASTQ format) were retrieved via Zenodo repositories corresponding to biological replicates of Basal and Luminal cell types (e.g., MCL1-DL, MCL1-DH).

	• Data Management: Utilized Galaxy Dataset Collections to manage biological replicates efficiently, ensuring parallel processing of samples and reducing manual file handling errors.
2. Quality Control (QC)
   
	• Tools: FastQC and MultiQC.

	• Objective: To assess sequencing quality prior to alignment.

	• Metrics Evaluated: Per-base sequence quality (Phred scores), GC content 	distribution, and adapter contamination levels.
	
	• Outcome: Identified read degradation at 3' ends and potential adapter 		read-through, necessitating a trimming step to prevent alignment bias.

3. Read Trimming & Artifact Removal
	• Tool: Cutadapt.

	• Parameter Configuration:

    	◦ Quality Cutoff: 20 (Phred score). Bases with a quality score below 20 were trimmed from the 3' end to ensure high-confidence mapping.
   
    	◦ Adapter Removal: Illumina Universal Adapters were scanned and removed.
   
    	◦ Minimum Length: Reads shorter than 20bp after trimming were discarded to reduce multi-mapping ambiguity.
   
4. Splice-Aware Genomic Alignment
   
	• Tool: HISAT2 (Hierarchical Indexing for Spliced Alignment of Transcripts).

	• Rationale: Unlike standard DNA aligners (e.g., Bowtie2), HISAT2 is splice-aware. It utilizes a graph-based FM index to accurately map RNA-seq reads across exon-exon junctions, accounting for the introns removed during mRNA processing.

	• Reference Genome: Mus musculus (mm10/GRCm38).

5. Gene Quantification
   
	• Tool: featureCounts.

	• Method: Mapped reads (BAM files) were overlapped with the reference gene annotation (GTF) to count reads mapping to exons.

	• Configuration:

    ◦ Strandedness: Unstranded (specific to library prep).
		
    ◦ Summarization: Multi-mapping reads were excluded to ensure quantification accuracy.
		
	• Output: A raw Count Matrix (Genes × Samples).

Downstream Statistical Considerations
The raw count matrix generated by this pipeline is not normally distributed and is influenced by sequencing depth. Therefore, prior to Differential Expression (DE) analysis, the data requires:
1. Filtering: Removal of lowly expressed genes (e.g., CPM < 0.5) to reduce statistical noise.
2. Normalization: Application of the Trimmed Mean of M-values (TMM) method. This calculates scaling factors to correct for library size differences and compositional bias between samples, ensuring valid biological comparisons in Limma-voom or edgeR.
